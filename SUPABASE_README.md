# Supabase Storage Setup \u0026 Guide\n\nThis project uses Supabase for storing and serving media files such as Images and PDFs. The integration is handled via the `@supabase/supabase-js` sdk in the backend.\n\nHere is a complete guide on how the storage buckets work and how to set them up.\n\n## 1. Supabase Dashboard Setup\n\n1. **Create a Project**: Go to the [Supabase Dashboard](https://supabase.com/dashboard) and create a new project.\n2. **Create a Bucket**: \n   - Navigate to the **Storage** section from the sidebar.\n   - Click **New Bucket**.\n   - Name your bucket (e.g., `pdfs` or `media`).\n   - Toggle **Public bucket** ON so that files can be successfully accessed via a plain public URL.\n3. **Set Bucket Policies** (Important):\n   - By default, Supabase storage operations are protected by Row Level Security (RLS).\n   - Since the backend uses the `Service Role Key` (`SUPABASE_SERVICE_KEY`), it **automatically bypasses** RLS upload/delete policies. \n   - Therefore, you don't have to create public policies for `INSERT`, `UPDATE`, or `DELETE`, ensuring your operations remain secure under backend control.\n   - Only `SELECT` operations require the bucket to be public for clients viewing the images/pdfs.\n\n## 2. Environment Variables (`.env`)\n\nIn your local `.env` file, ensure you have the following keys configured appropriately:\n\n```env\n# Found in Supabase -\u003e Project Settings -\u003e API -\u003e Project URL\nSUPABASE_URL='https://YOUR_PROJECT_ID.supabase.co'\n\n# Found in Supabase -\u003e Project Settings -\u003e API -\u003e Project API keys -\u003e `service_role` (secret)\nSUPABASE_SERVICE_KEY='eyJhbGciOi...'\n\n# The exact name of the bucket you created in the Storage dashboard\nSUPABASE_BUCKET=\"pdfs\"\n```\n\n## 3. How the Backend Code Works (`utils/supabase.js`)\n\nThe application includes utility functions to perform storage operations safely.\n\n### A. Uploading Files (`uploadFileToSupabase`)\n\nWhen a file (PDF or Image) needs to be uploaded, the backend calls this function:\n\n```javascript\nconst { uploadFileToSupabase } = require('./utils/supabase');\n\nconst publicUrl = await uploadFileToSupabase(filePath, {\n    fileName: 'invoice.pdf', \n    destinationPath: 'orders/invoice.pdf' // Will be placed in the 'orders' folder inside your bucket\n});\n```\n\n**Features:**\n- **Auto-Content-Type:** Automatically detects and applies `contentType` based on file extension (`.pdf`, `.jpg`, `.png`, `.webp`, `.gif`).\n- **Unique File Naming:** Prefixes uploaded files with a timestamp (e.g. `20261021_143000...`) so files are not accidentally overwritten.\n- **Returns URL:** Upon success, it fetches and returns the `publicUrl` directly, which you can save into your MongoDB database.\n\n### B. Deleting Files (`deleteFilesFromSupabase`)\n\nWhenever a record containing an image/pdf is deleted, it is essential to clean up the storage to free up space.\n\n```javascript\nconst { deleteFilesFromSupabase } = require('./utils/supabase');\n\n// You can supply an array of public Supabase URLs\nconst urlsToDelete = [\n    'https://YOUR_PROJECT.supabase.co/storage/v1/object/public/pdfs/orders/some_image.png'\n];\nawait deleteFilesFromSupabase(urlsToDelete);\n```\n\n**Features:**\n- The function intelligently slices the public URL to locate the internal path relative to the bucket, and deletes the referenced file directly using `remove()`.\n\n### C. Managing Storage Space (`getSupabaseStorageSize`)\n\nComputes the total byte size consumed by files in any target directory.\n\n```javascript\nconst { getSupabaseStorageSize } = require('./utils/supabase');\n\n// Get size of 'orders' folder\nconst sizeInBytes = await getSupabaseStorageSize('orders');\n```
